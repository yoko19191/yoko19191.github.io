哈希表(HashMap)
...

Note

哈希表，也被称为散列表, 字典，是一种非常高效的抽象数据结构，用于快速存储和检索数据。🔍 它的基本原理是通过一个叫做“哈希函数”的散列，将存储的数据（比如一个字符串或者一个数字）转换成一个整数，这个整数被称为“哈希值”。这个哈希值用作数组的索引，指向数据应该存储的位置。

想象一下你有一本电话簿，要快速找到某人的电话号码。如果电话簿是按字母顺序排列的，你可能需要翻很多页。哈希表就像是一个有特殊排列规则的电话簿，能让你快速找到你需要的号码。📖

哈希表的关键特点：

  1. 快速存取：理想情况下，哈希表能在接近常数时间内完成数据的存储和检索(avg,O(1); worst,O(n))，即操作的时间不随存储数据的数量增长而显著增加。
  2. 哈希函数：这是哈希表的核心，负责将输入（比如字符串或数字）转换成一个整数（哈希值）。一个好的哈希函数会尽量减少不同输入产生相同哈希值的情况，这种情况被称为“冲突”。
  3. 处理冲突：有时候，不同的输入会产生相同的哈希值，这就是“冲突”。常见的处理冲突的方法包括链地址法（在同一个哈希值下存储多个元素，形成一个链表）和开放寻址法（找到另一个空闲的位置来存储这个元素）。

🔑 举个例子：假设我们有一个简单的哈希函数，它将英文字母转换成对应的字母序号（A=1, B=2, ..., Z=26），然后将这个数值用作数组的索引。如果我们要存储单词“CAT”，哈希函数会计算出一个值（比如 3+1+20=24），然后我们就把“CAT”的信息存储在数组的第24个位置上。当我们需要查找“CAT”时，只需再次计算哈希值，然后直接访问那个位置，即可迅速找到所需信息。

在许多情况下，哈希表平均比搜索树或任何其他表查找结构更有效。因此，它们被广泛用于多种计算机软件中，特别是用于关联数组、数据库索引、缓存和集合。

0001. Two Sum | LeetCode Cookbook

遍历数组，使用哈希表存储已访问元素，快速检查每个元素的配对（即目标值减去当前元素的差值）是否已存在于哈希表中

class Solution:

	def twoSum(self, nums: List[int], target: int) -> List[int]:

		hash_map = {}

		for i, num in enumerate(nums):
			complement = target - num

			if complement in hash_map:

				return [hash_map[complement], i]

			hash_map[num] = i

		return []

Question

力扣（LeetCode）官网 - 全球极客挚爱的技术成长平台

给定一串字符串数组, 将字母异位词(包含字母相同但顺序不同)group起来并返回.

Check

两种解法.

第一种比较数组中经过排序过后的字符串是不是相等的, 如果是那就是Anagram就group起来.

  • 时间复杂度：O(nlogn)，因为排序操作通常需要 O(nlogn) 时间。
  • 空间复杂度：O(1) 到 O(n)，取决于使用的排序算法。

第二种是比较每个字符串中, 字符出现的次数. 如果相同那就是Anagram就group起来.

  • 时间复杂度：O(n)，只需遍历每个字符串一次。
  • 空间复杂度：O(1)，因为字符集的大小是固定的。
class Solution:
	# 排序字符串
	def isAnagrams(s, t):
		return sorted(s) == sorted(t)
		
	# 字符串计数
	def isAnagrams(s, t):

		if len(s) != len(t):
			return False

		count = [0]*26

		for i in range(len(s)):
			count[ord(s[i])-ord('a')] += 1
			count[ord(t[i]-ord('a'))] -+ 1

		for i in range(26):
			if count[i] != 0:
				return False

		return True

  

def groupAnagrams(self, strs: List[str]) -> List[List[str]]:

	anagrams_map = {} #用于存储每个已排序字符串及其对应的字符串列表。

	for s in strs:
		# 排序字符串作为键
		sorted_str = ''.join(sorted(s))
		if sorted_str not in anagrams_map:
			anagrams_map[sorted_str] = []
		anagrams_map[sorted_str].append(s)

	# 收集所有字母异位词分组
	return list(anagrams_map.values())

力扣（LeetCode）官网 - 全球极客挚爱的技术成长平台

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例 1：
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

示例 2：
输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9

Check

先将所有数字放到set里, 方便这样可以消耗常数时间检查一个数字是否在表中.

先检查每个数字num-1是否在数组中. 超出连续数字的起点.

然后检查这个起点数字的+1是否在表中. 如果在就count++

class Solution:

	def longestConsecutive(self, nums: List[int]) -> int:
		if not nums:
			return 0
	
		num_set = set(nums) # num_dict = {num: True for num in nums}
		max_length = 0
	
		for num in num_set:
		
			if num - 1 not in num_set:
				current_num = num
				current_length = 1

				while current_num + 1 in num_set:
					current_num += 1
					current_length += 1

				max_length = max(max_length, current_length)

		return max_length
Info

在 Python 中，集合（set）是一种无序、可变的数据结构，它类似于字典，但只包含键而没有对应的值。集合中的元素必须是唯一的。集合的实现基于哈希表，这使得元素的查找、插入和删除都具有很高的效率。

关于集合的一些特点：

  • 无序性：集合中的元素没有固定的顺序。
  • 唯一性：集合中的元素是唯一的，重复的元素会被忽略。
  • 可变性：集合是可变的，你可以添加或删除元素。

在解决问题时，使用集合可以提供快速的成员检查，因为集合的查找操作的平均时间复杂度是 O(1)。这对于问题中需要频繁检查元素是否存在的场景非常有用。而字典是基于集合实现的，因此也具有类似的性能特点。

0283. Move Zeroes | LeetCode Cookbook

最简单的双指针题, 直接用两个指针, 一个指零一个指非零元素, 然后swap. 类似只遍历一遍的冒泡排序.

class Solution:

	def moveZeroes(self, nums: List[int]) -> None:
		"""
		Do not return anything, modify nums in-place instead.
		"""
		zero_pt = 0
	
		for i in range(len(nums)):
			if nums[i] != 0:
				nums[i], nums[zero_pt] = nums[zero_pt], nums[i]
				zero_pt += 1

11.盛最多水的容器(Container with most water)

Question

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

说明：你不能倾斜容器。

你可以想象在一个平面上有一些垂直放置的线段，每个线段的高度由数组中的元素值表示。这些线段可以看作是围成容器的边界，而容器的底部是 x 轴。题目的目标是找到两个线段，它们与 x 轴构成的容器可以容纳最多的水。

例如，考虑数组 [1,8,6,2,5,4,8,3,7]。我们可以选择将容器的边界放在 1 和 7 这两个元素上，它们之间的距离为 8−1=7，高度为 min(1,7)=1，因此容器的容量为 7×1=7. 这是一种方式，我们可以尝试移动边界，寻找更大的容器。

这个问题的本质是在数组中找到两个高度，使它们之间的距离最大，同时高度较小的那个决定了容器的容量。通过不断移动较小高度的边界，我们希望找到最大的容器容量。

Check

0011. Container With Most Water | LeetCode Cookbook

初始化左指针在index=0, 右指针index在末尾. width = right - left, height = max(height[left], height[right]). 计算area. 哪边指针指向的元素小哪边的指针就往哪边靠向中间. 直到两指针相遇.

class Solution:

	def maxArea(self, height: List[int]) -> int:
		left, right = 0, len(height) - 1
		max_area = 0
		
		while left < right:
			area = min(height[left], height[right]) * (right - left)
			max_area = max(max_area, area)
		
		  
		if height[left] < height[right]:
			left += 1
		else:
			right -= 1
		
		return max_area

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请
你返回所有和为 0 且不重复的三元组。(找出所有和为0的三个元素的组合)

注意：答案中不可以包含重复的三元组。

示例 1：

输入：nums = [-1,0,1,2,-1,-4]
输出：-1,-1,2],[-1,0,1
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。

示例 2：

输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。

示例 3：

输入：nums = [0,0,0]
输出：0,0,0
解释：唯一可能的三元组和为 0

0015.3 Sum | LeetCode Cookbook

其实是用三个指针和先对数组排序.
外层套一个循环便利排序过后的数组.
用两个指针高效夹逼寻找第一个元素的补数.
然后在去除第一个元素和两个指针前后的重复元素以防止输出重复的三元组

class Solution:

	def threeSum(self, nums: List[int]) -> List[List[int]]:
		ans = [] 
		nums.sort() # nlogn

		for i in range(len(nums)):
			#检查指向的第一个元素是不是重复的
			if i > 0 and nums[i] == nums[i-1]:
				continue

			target = -nums[i]
			left, right = i-1, len(nums)-1 #因为sorted了所以left可以从i左边开始

			while left < right:

				if nums[left] + nums[right] == target:
					ans.append([nums[left], nums[i], nums[right]])

					#去除left, right指向的元素是不是重复的
					while left < right and nums[left] == nums[left+1]:
						left += 1
					while left < right and nums[right] == nums[right-1]:
						right -= 1
					#找到和为零的时候移动两个指针
					left += 1
					right -= 1
				#意味着左边的数太小了凑不上target
				elif nums[left] + nums[right] < target: 
					left += 1
				#意味着右边的数太大了超过了target
				else:
					right -= 1
					
		return ans

Hint

堆（Heap）：LIFO + 函数内部变量

  1. 后进先出（LIFO）： 栈是一种后进先出的数据结构，即最后压入栈的元素最先弹出。
  2. 固定大小： 栈的大小通常是固定的，分配的内存在编译时就确定。
  3. 快速访问： 栈的操作包括压入（push）和弹出（pop），这两个操作的时间复杂度都是 O(1)。
  4. 局部变量存储： 函数调用时，局部变量会被存储在栈上，函数调用结束后会自动释放。

PS: 堆上分配的内存需要手动释放(C中free和C++中的delete)，或者由GC(Java, Python)处理, 以避免内存泄漏和野指针等问题。

Hint

栈 (Stack) LIFO + 手动内存管理

  1. 大小固定或有限制：栈的大小通常在程序启动时就已经确定，且空间相对较小。
  2. 后进先出 (LIFO)：数据的存取遵循后进先出的原则，最后放入栈的元素将会是第一个被取出。
  3. 快速访问：栈上的数据访问非常快速，因为它使用指针（如堆栈指针）来直接访问。
  4. 自动管理：由编译器自动管理分配和释放，不需要手动干预。
  5. 局部变量和函数调用：主要用于存储局部变量以及记录函数调用时的返回地址和参数等。所以生命周期较短, 会随着函数结束而结束
  6. 无碎片问题：在栈上分配内存时不会产生内存碎片问题。
  7. 线程独占：每个线程通常都有自己独立的栈空间。
