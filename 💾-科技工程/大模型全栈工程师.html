我对LLM的理解

我的理解是, LLM训练的人类文本, 包含了人类的直接观察(眼耳鼻舌身意)和各种层级的抽象(受想行识) (有点像训练数据本身就有一个U-Net+Skip-connection)导致的结果是文本涵盖更多范围的世界的碎片(相比图像数据库). 能够让大参数神经网络知识抽象能力和压缩能力的得到实际的效用.

LLM从文本学习到了世界知识的一个压缩或者抽象, 依赖Transfomer 抽象人类很难想到的高纬度的非平凡的规则. LLM拥有的知识广度和深度使得跨学科跨领域的信息整合能力成为可能. 比如, 如果研究“文化演变” 大多数人类可能会从历史, 社会科学和人类学的角度思考. 假设在讨论中引入了量子计算机的发展。一般人可能很难立即看到量子计算和文化演变之间的直接联系。但是LLM可以识别出这样一个抽象的联系：量子计算机作为一种高科技产品，其发展不仅仅是技术的进步，也反映了人类文化对于高科技和理论物理的接受和理解的演变。此外，量子计算机的潜在能力可能会影响信息处理、加密技术甚至日常生活，这些改变最终会深刻影响社会结构和文化价值观。

我们此时确定LLM有能力更宏观、更深入的角度输出高度抽象的概念和现象, 但为什么LLM在实际使用中还经常输出片汤话呢? 我的即使有 tempeture 参数控制选择候选词的随机性, 但是依然倾向于输出它抽象最中心, 期望最高的数据. 有点类似于“如果一个人知道所有东西, 那么就意味着它什么都不知道”. 我们可以从头开始训练一个小范围的特化的语言模型(这耗时耗力而且小公司也没有资源训练), 也可以用直接l利用GPT这类大模型通用能力有效应对多样化、碎片化的AI应用需求. 而Prompt提供先验信息或者我更加乐意描述为手动屏蔽LLM高维度抽象的特定方向(因为足够的大模型应该已经包含所有的Prior)已满足实际的工程需求. 类似于让一个全能干细胞经过发育(Fine-tuning)调控(Prompt, Custome Instruction, Embedding, Fine-tuning)分化成不同的体细胞. 这也是Prompt工程师的价值所在. 这篇论文现实Prompt的效用即使是在参数量小的多的BERT上也有体现.

LLM的特点, 局限和优化方向

局限&机会

  • @ 泛化能力缺陷(无法对训练数据进行全面有效的泛化)
    • 反向推理差
      • 虽然大模型在大量文本数据的训练上展现了惊人的泛化能力. 但在泛化抽象逻辑关系和推理新因果关系依然有局限, 因为训练数据不包含精准的逻辑和因果框架
      • 反向推理, “所有猫是动物”（A是B）是正确的，但是“所有动物是猫”（B是A）”是错误的. LLM在理解这样拥有非对称逻辑(简单颠倒因果命题不为真)关系时可能有很大的局限. 因为理解因果关系比单纯的关联关系要复杂得多.
    • 负面知识处理有问题
  • @ 事实性缺陷(模型缺乏相对应的数据)
    • 实时性知识差
    • 缺乏领域知识
  • @ 统计模型自身(由模型原理自身带来的缺陷)
    • “幻觉”(Hallucination, the generated content that is nonsensical or unfaithful to the provided source content)为什么大模型会「说胡话」？如何解决大模型的「幻觉」问题？ - 知乎
    • LLM缺乏可解释性, 无法预测它的输出(不能保证LLM不会产生偏见、误导、攻击、伤害用户的表述)
  • ! 大模型优化方向
    • 减少训练的开销
    • 优化训练的难度
      • 由于大模型的参数量和数据激增, 训练中所产生的静态内存(模型变量)、动态内存(中间变量, BP, 隐藏层参数通常是静态内存的4倍)和计算的模型误差的巨大的算力要求, 使得大模型的分布式训练(按照并行对象可以分为数据并行, 模型并行, 流水线并行)变得必不可少. 大模型的机内高速通讯,机间低延迟高吞吐, 同时GPU集群要支持月级无错误无中断的训练. 其成本、数据、人才、算法、框架的全流程支持和相互协作可想而知的难.
      • 数据并行(最容易): 计算集群里的每个节点获得数据子集, 后对其单独进行一次误差计算, 最后当所有节点step结束后, 对误差结果进行平均或者其他更精细的操作. 优点是节点间的通讯要求低, 并行效率高, 可以有单机failure的容错机制. 缺点是只能训练, 无法推理.
      • 模型并行(aka. 网络并行): 计算集群中的每个节点共享相同的数据集和模型参数, 但是每个节点只会训练模型中的一部分, 由此将计算负载和内存负载切分到多台节点上. 缺点是通讯和计算是串行的, 对通行的要求极高.
      • 流水线并行: 计算集群中每个节点各自对模型的不同层进行误差计算. 优点是对通信的吞吐量要求低. 缺点是对通信的延迟要求高, 容易造成流水线间的空闲.

    • 减少推理的开销
      • 降低量化精度(FP32->INT8); 稀疏化; 剪枝法

计算不可约性（Computational Irreducibility）是一种由Stephen Wolfram提出的概念，用于描述某些系统的行为是不可预测的，且唯一确定其未来状态的方法就是实际进行模拟或计算该系统。

e.g.
想象一场森林火灾的扩散。火灾如何扩散依赖于树木的分布、风的方向、湿度等复杂因素。预测这场火灾的具体行为几乎是不可能的，唯一的方法是通过模拟或实际观察其扩散过程。即便有基本的火灾扩散原理，每一场火灾的具体细节和最终结果都是独特的，不能简单地通过已知公式或规则预测。

LLM对知识, 对高维数据, 强大的抽象总结能力, 表示那些原本人类以为只有人可以干的事情, 比如写作, 比我们想象的要简单, 有规律的多. 但是, 由于计算不可约性的存在, 复杂系统比如天气,人类社会,甚至股票市场, 对他们规则的总结总是有一个上限. 还是需要通过实际观察或者模拟才能够得倒某些结论. 特别对于像GPT这类Decoder-only的网络结构, 输出Token只经过前馈网络, 更加不具有通用的推理计算能力. 对于那些需要通用计算的场景, GPT4内置一些基础的代数运算可以满足简单计算. 但是更多时候, 依赖类似AutoGPT或者Code Interpertuer 这样的外挂模块实现. 可以说现在的LLM不具有通用的计算能力.

同时, 计算不可约性的存在, 也意外着像科学发现, 艺术创造等不具有可约性的系统, 永远有收获惊喜的能力. 人类和LLM在这方面是相同.下面我理解的ChatGPT的长处和短板

出家人可以在狼人杀中扮演狼人么?

在探讨这个问题之前，让我们首先了解一下“狼人杀”这个游戏。狼人杀是一种角色扮演游戏，玩家被分配到不同的角色，比如村民、狼人等，每个角色都有其特定的目标和规则。游戏的核心在于策略、沟通和推理。

接下来，我们来看看出家人的角色。在许多文化和宗教中，出家人是追求精神生活、修行和宗教实践的人。他们通常遵循一套严格的道德规范和生活准则。

将这两者结合起来考虑，我们可以发现，这个问题的核心在于角色扮演游戏中的虚构角色和现实生活中的道德准则之间的关系。在狼人杀这样的游戏中，玩家扮演的角色与他们在现实生活中的身份是分离的。游戏中的角色和行为是虚构的，不反映玩家的个人道德观或现实身份。

因此，如果出家人参与狼人杀游戏，从游戏的角度来看，他们扮演狼人或任何其他角色都是可以的。这种角色扮演不会影响他们在现实生活中的道德和信仰。然而，重要的是要考虑到个人的舒适度和信仰对于某些类型娱乐活动的限制。如果出家人因为个人信仰或道德原则而不愿意参与这类游戏，这种选择也应得到尊重。

下面我列举了LLM的局限性:

  1. LLM基于统计和模式匹配，而非真正的理解或意识. 所以理解和推理能力是有限的
  2. 只能模仿情感表达，但它并不真正理解或体验情感。因此，在需要深层次情感理解和真诚情感表达的场合，LLM的回答可能显得机械或不贴切
  3. 在需要人类特有的直觉和感知的任务(e.g.如艺术创作、情感察觉)它的表现可能不及人类
  4. 有时会写出看似合理但不正确或荒谬的答案。
  5. 对输入措辞的调整或多次尝试相同的提示很敏感。例如，给定一个问题的措辞，模型可以声称不知道答案，但只要稍作改写，就可以正确回答。
  6. 回答通常过于冗长并过度使用某些短语。
  7. 对于模棱两可的问题，模型通常会猜测用户的意图，而非让用户澄清问题。·模型有时会响应有害的问题或表现出有偏见的行为。
  8. 在数学和物理等需要进行数字推理的任务中仍然会出现一些错误。

LLM 原理

Abstract

用不严密但通俗的语言描述大模型的工作原理：

  1. 大模型阅读了人类曾说过的所有的话。这就是「机器学习」，这个过程叫「训练」
  2. 把一串 token 后面跟着的不同 token 的概率存入「神经网络」。保存的数据就是「参数」，也叫「权重」
  3. 当我们给它若干 token，大模型就能算出概率最高的下一个 token 是什么。这就是「生成」，也叫「推理」
  4. 用生成的 token，再加上上文，就能继续生成下一个 token。以此类推，生成更多文字

语意空间

这就是ChatGPT

  • ? 什么是语义空间(Semantic hyperspace)
  • “语义(Semantics)” 指词、短语或语句的意义和解释
  • 🔍语义空间的定义: 语义空间 是一种用来表示和处理词汇、短语或文本的意义数学或计算模型. 在这个假想的空间中，每个词或短语都被表示为一个向量。这些向量捕捉了词汇之间的关系和属性，比如它们的相似性和差异性。
  • 🔢如何构建语义空间：构建这样的空间通常需要大量的文本数据。通过分析这些文本，可以学习到词汇之间的共现（co-occurrence）关系和上下文信息。常见的方法包括词袋模型（Bag of Words）、TF-IDF（Term Frequency-Inverse Document Frequency）和词嵌入（Word Embeddings）如Word2Vec、GloVe等。
  • 🌌 语义空间的作用：在语义空间中，可以执行各种有趣的操作。例如，可以计算两个词向量之间的距离来衡量它们的语义相似度。这种表示还可以用于各种NLP任务，如文本分类、情感分析、机器翻译等。
  • 💡 一个例子：想象一下，“猫”和“狗”在语义空间中的向量会彼此非常接近，因为它们都是宠物、都是动物。但是，“猫”和“汽车”的向量就会相距较远，因为它们属于完全不同的类别

Prompt 典型构成

介绍两个用于提升语言模型输出质量的Prompt工程框架:

Elavis Saravia框架：

  • Instruction(指令)：明确模型需要执行的特定任务, 如文本生成、翻译语言和创作不同类型的内容.
  • Context(上下文)：为模型提供理解请求所需要的背景信息. 例如, 为一个查询中国历史的AI提供25史的文本；
  • Input Data(输入数据)：模型需要处理的具体数据.
  • Output Indicator(输出数据)：显式要求输出的类型或者格式. 例如, 制定输出为JSON格式.

CRISPE 框架：

  • Capacity and role(能力和角色)：定义期待AI扮演的角色. 比如心理学家、哲学教授或者政治家
  • Insight(洞察)：提供模型理解请求的背景信息和上下文
  • Statement(声明)：明确模型需要执行的特定任务
  • Personality(个性)：定义模型回答请求的风格或者方法. 比如, 猫娘, meow
  • Experiment(实验)：通过提供多个答案请求来迭代, 以获得更好的答案.

我们可以看到两个模型都强调了Prompt要清晰, 上下文的重要性和指定输出方式的能力.

  • @ 角色：给A!定义一个最匹配任务的角色，比如：「你是一位软件工程师」「你是一位小学老师」
  • @ 指示：对任务进行描述
  • 上下文：出与任务相关的其它背景信息（尤其在多轮交互中）
  • @ 例子：必时给出举例，学术中称为one-shot learning,few-shot learning或in-context learning; 实践证明其对输出正确性有很大帮助
  • @ 输入：任务的输入信息；在提示词中明确的标识出输入
  • @ 输出：输出的格式描述，以便后继模块自动解析模型的输出结果，比如(JSON、XML)

Note

介绍了思维链的方法.

  • Few Shot: 在Prompt中给出符合context的example

如果Few Shot不行

  • Few Shot COT(Chain of Thought): 在example中包含思考和推理的细节

如果没有合适的Example
那么就在Prompt最后添加“Let's think step by step”

Self-Consistency : 或者用同一个Prompt问多次, 然后手动挑选一个(e.g. AutoGPT)

或者将一个任务分割成多个小任务

Generated Knowledge: 即在让LLM产生最终的答案前，先通过第一步prompt让LLM产生一些对回答最终问题有潜在帮助的有用的信息，然后再附带这些信息作为context给LLM提第二步prompt。

  • @ 思维链 (Chain of Thought, COT)
    • 思维链，是大模型涌现出来，偶尔发现的一种能力
    • 有人在提问时以「Let’s think step by step」开头，结果发现 AI 会把问题分解成多个步骤，然后逐步解决，使得输出的结果更加准确。
    • 原理可能是 AI 生成更多相关的内容，构成更丰富的「上文」，从而提升「下文」正确的概率
    • 这对涉及计算和逻辑推理等复杂问题，尤为有效
    • 案例：客服质检，用于检查客服与用户的对话是否有不合规的地方
  • @ 思维树（Tree-of-thought, ToT）
    • 在思维链的每一步，采样多个分支
    • 拓扑展开成一棵思维树
    • 判断每个分支的任务完成度，以便进行启发式搜索
    • 设计搜索算法
    • 判断叶子节点的任务完成的正确性
    • 案例：指标解读，项目推荐并说明依据，小明 100 米跑成绩：10.5 秒，1500 米跑成绩：3 分 20 秒，铅球成绩：12 米。他适合参加哪些搏击运动训练。
    • ! 只有GPT-4目前才能跑动思维树
  • @ 自洽性（Self-Consistency）
    • 一种对抗「幻觉」的手段。就像我们做数学题，要多次验算一样
    • 通过同样的Prompt跑多次，通过投票选出最终结果

Embedding

Embedding 模型

Embedding 是将文字转换成用向量表示的一种技术，方便描述为文本之间的相似度：

  • 搜索（根据查询字符串的相关性进行排序的结果）
  • 聚类（根据相似性将文本字符串分组）
  • 推荐（根据具有相关文本字符串的项目进行推荐）
  • 异常检测（识别具有较少相关性的异常值）
  • 多样性测量（分析相似性分布）
  • 分类（根据最相似的标签对文本字符串进行分类）

Vector Database

What is a vector database? | Cloudflare

  • 什么是向量数据库?
    • 向量数据库 是一种专门存储高维度数据数据的数据库. 可以为AI应用的高维度信息提供快速高效低成本的增删改查. 允许开发者更加高效的利用日益增长的非结构的数据.
    • 和传统观念数据库相比, 向量数据库有如下优势
      • 充分利用语义上下文信息, 赋能AI时代应用场景(e.g. 图片的模糊匹配)
      • 更加高效的索引和搜索. 向量量数据库使用特殊的索引结构（如KD树、R树）和算法（如近似最近邻搜索），能够在高维空间中高效地执行搜索操作
      • 快速的相似度计算：它们针对高维数据的相似度计算进行了优化，如余弦相似度或欧几里得距离，这是AI应用中常用的度量方式
      • 扩展性和灵活性：向量数据库通常设计得更加灵活和可扩展，能够处理大量动态变化的AI数据集. 比如, LLM + RAG + 向量数据库, 可以利用 LLM 处理企业私有数据.
      • 多模态数据存储: 允许文本、视频、照片、语音等多种模态的共同存储.

LoRA

作者亲自讲解：LoRA 是什么？_哔哩哔哩_bilibili
LORA：大模型轻量级微调 - 知乎

Low Rank Adaptation (LoRA) 是一种在不牺牲推理速度下，通过训练很少参数和保留原始性能定制类似扩散模型，大型语言模型的技术。从而满足公司实际业务场景和垂直领域需求的方案。

在LoRA诞生之前，主流的方法包括2019年 Houlsby N 等人提出的 Adapter Tuning，2021年微软提出的 LORA，斯坦福提出的 Prefix-Tuning，谷歌提出的 Prompt Tuning，2022年清华提出的 P-tuning v2。但它们各自有一些问题。

  • Adapter Tuning 增加了模型层数，引入了额外的推理延迟
  • Prefix-Tuning 难于训练，且预留给 Prompt 的序列挤占了下游任务的输入序列空间，影响模型性能
  • P-tuning v2 很容易导致旧知识遗忘，微调之后的模型，在之前的问题上表现明显变差

Modeling Editing

Cite

大语言模型的知识编辑：问题，方法与挑战_哔哩哔哩_bilibili
[2305.13172] Editing Large Language Models: Problems, Methods, and Opportunities

  • 介绍了不同于Fine-tuning, RA 之外, 建立的的基准数据集, 评估了几种LLM编辑方法(特别是generalization和efficiency)并对他们未来的研究潜力进行了展望
  • LLM Editing
    • 定义: 使用一定方法对一定示例范围内的模型行为进行修改, 而不改变范围外模型的性能. 修改后的模型需要满足 Reliability, Generality, Locality 三个特性. 以满足LLM的准确性和及时性, 并对有偏回答进行修正.
    • 修改路径1: 冻结原模型层同时添加额外参数(Memory-based)
      • SERAC Mitchell 等人(2022b) 外挂一个范围分类器来评估存储编辑示例范围内的新输入的可能性, 如果输入与缓存中的任何编辑对齐，则获取具有最高概率的编辑，并根据输入和检索到的编辑返回反事实模型的预测
      • T-Patcher Huang 等人(2023)和 CaliNET Dong 等人(2022)将可训练参数引入 PLM 的前馈模块, 冻结原模型后在修改后数据上的训练新引入的参数
    • 修改路径2 : 修改模型的内部参数
      • Locate-then-edit:  The Knowledge Neuron (KN) method Dai et al. (2022a)介绍了一种知识归属技术，以精确定位“知识神经元”(前馈网络(FFN)矩阵中的一个关键值对) ，这个关键值对体现了知识，然后继续更新这些神经元.ROME Meng 等(2022)应用因果中介分析来定位编辑区域。ROME 不修改 FFN 中的知识神经元，而是改变整个矩阵。但是，KN 和 ROME 一次只能编辑一个事实关联。，MEMIT Meng 等人(2023)被提出可以修改一系列层次，并促进数以千计的改变被同时执行。请注意，这些方法是基于事实知识的局部性假设，尚未得到广泛的验证，某些参数的变化可能会影响不相关的知识，并产生不可预见的结果。
      • Meta-learning: 元学习方法使用一个超网络来学习编辑语言模型(LM)所必需的参数. 知识编辑(KE) De Chao 等人(2021)利用超网络(具体而言，是一个双向的 LSTM)来预测每个数据点的权重更新，从而使编辑目标知识的约束优化不会干扰其他人。但是，在编辑LLMs时，此方法不足。为了克服这个限制，使用梯度分解(MEND) Mitchell 等人(2022a)的模型编辑器网络能够使用单个输入输出对对语言模型进行有效的本地编辑。具体来说，MEND 学习通过使用梯度的低秩分解来转换经过微调的语言模型的梯度，这种方法可以以最小的资源开销应用于LLMs。与先定位后编辑的方法相比，元学习方法需要额外的训练阶段，这可能导致时间和记忆成本的增加。
  • Evaluating Methods:
    • Dataset: ZsRE(QA数据集) 和 CounterFact
    • Evaluated model:  T5-XL (3B) (encode-decode) 和 GPT-J (6B) (decode-only)
    • 评价维度
      • Portability, 衡量知识转移到相关内容的有效性. （1）Subject replace，（2）Inverse relation（3）One-hop Reason
      • Generality(success rate within scope)
      • Locality(control editing within scope)
      • Efficiency(time, gpu, memory)
  • 发现&结论
    • 现有LLM编辑技术在某些方面表现出色，但在可移植性、局部性和效率方面仍有显著的提升空间。这些技术在处理知识更新时，往往难以确保知识在相关上下文中一致且连贯地反映出来。
    • 由于计算资源的限制，研究主要集中在20亿参数规模的模型上。对于不同架构, 更大参数的模型，如Llama，这些编辑方法的有效性尚待探索。
    • 随着多语言编辑的需求日益增长，这是一个值得未来关注和探索的重要研究方向。
    • 对于那些仅通过API可访问的“黑盒”模型，如ChatGPT和GPT-4，如何进行有效的编辑仍然是一个挑战。当前的一些方法，如使用上下文学习和基于提示的方法，显示出了在模型编辑方面的潜力。
    • 论文指出了LLMs编辑领域的一些开放问题，如多跳推理设置、知识注入和更新的基准测试，以及从LLMs中删除知识或信息的能力

Tempeture Controls randomness lowering results in less random completions. As the number approaching zero, the answee will become determinstic and repetive.

Maximum length the maximum number of token to shared between the prompt and completion. The exact number various by models. (One token is roughly 4 characters for standard english text)

Stop sequences up to four sequences where the API will stop generate further tokens. the returnd text will not contain the stop sequences.

Top p 是一种生成文本的抽样策略. p是概率阈值我们按照每个词语生成的概率从高到低排序，直到累加的概率达到或超过p. 只考虑最有可能的那些词语. 以帮助LLM在保持回答多样性的同时, 避免选择到极小概率的词语.

Frequency Penalty 是指在文本生成过程中，降低已经出现过词语的重复出现概率的参数.

Presence Penalty 是指在文本生成过程中，减少已经出现过的概念或主题重复出现的概率的参数。鼓励模型探索新的想法或话题，使对话更加深入和广泛而不是对同一个主题原地打转.

向量数据库（Vector Database）：这是一种专门为存储向量数据（如嵌入向量）设计的数据库。在这个数据库中，数据通常是高维空间中的点，它支持高效的相似性搜索，也就是说，可以快速找到与查询向量最相似的向量。

向量检索（Vector Search）：这是一种检索技术，用于在向量数据库中快速找到与给定查询向量最相似的项。这通常涉及到使用余弦相似度或欧氏距离等度量来比较向量之间的相似性。

Agent：在人工智能中，Agent通常指的是能够感知环境并作出反应以实现特定目标的实体。在多Agent系统中，多个Agent可能会互动以完成更复杂的任务。
